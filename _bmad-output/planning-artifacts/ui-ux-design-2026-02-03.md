# UI/UX Design Document: SDLC Game Dashboard

## 1. Design Overview

### 1.1 Design Philosophy

| Principle | Description |
|-----------|-------------|
| **Game-First** | Dashboard should feel like a game, not a monitoring tool |
| **Clarity** | Agent activities must be instantly understandable |
| **Delight** | Cute chibi characters create emotional connection |
| **Minimal UI** | Let the game view dominate, UI should be unobtrusive |

### 1.2 Visual Identity

**Tagline:** *"Looks like a game, but it's real AI at work"*

**Art Style:** Chibi/Kawaii isometric 2.5D
- Big heads, small bodies (2:1 ratio)
- Soft, rounded shapes
- Vibrant but not harsh colors
- Subtle shadows for depth

**Mood:** Professional yet playful, like Theme Hospital meets Slack

#### Auto Color Generation Algorithm

```typescript
// When all predefined colors are used, generate deterministic color from role name
function generateRoleColor(roleId: string): string {
  // Hash the role ID to get consistent color
  const hash = hashString(roleId);

  // Generate HSL color with:
  // - Hue: Based on hash (0-360)
  // - Saturation: 70% (vibrant but not neon)
  // - Lightness: 50% (balanced)
  const hue = hash % 360;
  return `hsl(${hue}, 70%, 50%)`;
}
```

## 9. Accessibility Considerations

### 9.1 Color Contrast

| Element | Foreground | Background | Ratio |
|---------|------------|------------|-------|
| Primary text | #F8FAFC | #1E293B | 15.8:1 âœ… |
| Secondary text | #CBD5E1 | #1E293B | 9.7:1 âœ… |
| Agent labels | #FFFFFF | Role color | Varies (add shadow) |

### 9.2 Visual Indicators

- Never rely on color alone
- Agent status shown via animation + icon + color
- Selected state has ring + tooltip (not just color change)

### 9.3 Text Readability

- Minimum font size: 12px
- Agent labels: 14px bold with shadow
- Log entries: 13px monospace
- Zone labels: 16px bold
